elmのインストール

　一番最初にやることって言ったら、まずは処理系と環境のインストールよね。
ちなみにわたしが使ってるArchLinuxでのインストール方法は…… 公式によると$ sudo npm install -g elmで入るらしいけど、permission errorで入らなかったから…… yarnで入れたわ。

$ sudo yarn global add elm

　これによって、以下の4つのパッケージが入ってくるわ。

  • elm-repl
  • elm-reactor
  • elm-make
  • elm-package

elm-replがREPL、 elm-reactorがプロジェクトランナー、 elm-makeがJavaScript（or HTML）へのトランスパイラ、 elm-packageがelmのパッケージマネージャなのね。

REPLの起動

　やったー！ elmがインストールできたー！

　まずは焦らずに… 公式の手引き通り、replを起動するわ。

$ elm repl
> 1 + 2
3 : number

> succ n = n + 1
<function> : number -> number
> succ 0
1 : number

> succ = \n -> n + 1
<function> : number -> number

> True
True : Bool
> False
False : Bool

> prev n = \
|   n - 1
<function> : number -> number

> [1, 2, 3]
[1,2,3] : List number

> (+)
<function> : number -> number -> number

うまくいったわね！

ちなみに以下の実行たちは失敗したわ

  • > 1 : Float
    • Function signature - Issue #86 - elm-lang/elm-repl - GitHub
    • にこ、そこまで英語とくいじゃないけど、このissueを読んでみたところ…なるほど。elm-replでの型アノテーションは付けられないのね
  • > x' = 1
    • 変数名に'は許可されないんだ…
  • > [1..3]
    • ..構文はないみたね。代わりはありそうだけども
  • > toFloat 1 + List.length [1,2,3]
    • 「FloatとIntは足し合わせられない」という旨のエラーが出たわ
    • ちゃんと型クラスがなくても、+に曖昧性はないみたい。しっかりしてるわね！

.elmの実行

　にこ、ちょっと調べてみたんだけど、.elmの実行に際しては elm-makeでJavaScriptにコンパイルしてconsole.logとかするよりも、 HTMLにコンパイルしてDOMに文字列を表示したりする方がスタンダードっぽそうね。

　とりあえずこんな感じで実行できたわ。

Main.elm

import Html exposing (Html, div, text)

nico : { x : number, y : number1 }
nico = { x = 74, y = 17 }

main : Html a
main =
  let mine = nico.x
  in div [] [toString mine |> text]
$ elm-make Main.elm --warn --output /tmp/elm.html
$ $BROWSER /tmp/elm.html

って誰が74よ！

……？

  • nico : { x : a, y : b }
    • nicoの型をこんなのにしたら、ちゃんとtype mismatchのエラーが出たわ
  • nico : { x : number, y : number25252 }
    • こんな感じにしてもコンパイルが通るから、number{n}って形はNum a => aみたいに見做されるのかしらね
  • mine |> y |> toString |> text
    • 「yは変数/関数でない」旨のエラーが出たわ。Elmのレコードは関数にはならないのね
